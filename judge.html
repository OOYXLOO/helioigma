Playability proof

Helioigma is built to feel like a short, finishable jam game before it asks for verification. The receipt layer supports judging, but the player loop stays readable: choose a node, see the wheel react, decide whether to chase the next mismatch or spend a Hint, then recover cleanly if nightfall wins.

Readable decisions

Every phase shows the target glyphs, aligned-node count, and phase-specific scan rule, so each tap has a visible reason.

Immediate feedback

Node pulses, Rotor Trace, daylight meter movement, optional audio, and status copy make correct and incorrect shifts legible without reading source code.

Finished failure state

Nightfall produces a report with held phases, score, shifts, and a direct retry or Demo Solve route instead of dropping the judge into a dead end.

Ode with restraint

The Turing reference is not meant as decoration or biography. The player makes a hypothesis about the rotor, tests it with a move, watches state change, corrects the next mismatch, and finishes with evidence that can be checked. That is the tribute Helioigma can honestly make in a small browser game: reasoning as play, and evidence as something visible.
