Run receipt verifier

Check a Helioigma completion receipt.

Completed runs expose a summary receipt in the form SC-4P-score-shifts-checksum. This page recomputes the checksum locally so judges can tell whether the receipt matches the published summary formula, and whether it is the exact stable one-click Demo Solve receipt.

Formula: solstice|4|score|shifts|4 is hashed with FNV-1a and converted into a six-character base-36 suffix.

Source note: the receipt is generated in game.js by buildRunProof, and both the game and verifier use the shared public receipt-core.js checksum formula.

Stable demo boundary: only SC-4P-2907-62-Y5VFX1 is marked as the published stable Demo Solve receipt. Other formula-valid receipts are shown as neutral checksum-form matches, not as the published demo route.

This is a transparent summary receipt for a public jam entry, not anti-cheat, identity, payout, or eligibility proof. It exists so judges can replay the public Demo Solve route and inspect the score, shift count, phase count, and checksum without trusting a screenshot.

What this checks

The stable receipt matches the published four-phase summary payload shape, score, shift count, solved-phase count, and local checksum formula for the public Demo Solve route.

What this does not check

It does not check player identity, anti-cheat integrity, payout eligibility, or that a private account action happened.

Sample payload: solstice|4|2907|62|4 produces suffix Y5VFX1 for receipt SC-4P-2907-62-Y5VFX1.
